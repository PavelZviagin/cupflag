# WRITEUP - LVL1

## Запуск

```bash
python run_lvl1.py --duration 300
```

Флаги логируются с миллисекундными метками в stdout и дописываются в
`flags_lvl1.jsonl` (дедупликация, decoy-флаги отбрасываются).

## Архитектура решения

Код - в `cupflag/lvl1/` (`config.py` + `runner.py`); общий каркас (`BaseRunner`,
логирование, sink флагов, HTTP-сессия) - в `cupflag/common/`. `Lvl1Runner`
наследует `BaseRunner` и задаёт только своё: вход (логин), рабочую петлю и строку
статистики.

**Протокол.** Аутентификация: `POST /login` с `login=<user>&password=md5(user)`
-> cookie `sid`. Захват: `POST /v1/capture` с телом `{}` возвращает
`{"status":"closed"}`, `{"status":"ok","flag":"CUPFLAG{…}"}` или
`{"status":"rate_limited","retry_after_ms":N}`.

**Ключевая находка.** Окна открываются строго на **границах целых секунд**
серверных часов (подтверждено: все захваты попадают в коридор +17…+34 мс после
целой секунды). Промежуток между окнами - случайное целое 5–14 c. На окно - один
флаг; расписание **глобальное** (два аккаунта получают идентичные флаги).

**Стратегия - RTT-таймированный залп на границе.** На каждой границе секунды
раннер даёт короткий залп из `shots` (по умолчанию 2) запросов, рассчитанный так,
чтобы они **долетели в окно**:

- каждый `POST /v1/capture` засекается (даже `closed`), питая EWMA от RTT;
- центр залпа = `target_arrival − RTT/2` (целимся ~50 мс в окно), выстрелы
  расходятся на `span` вокруг центра - это гасит асимметрию задержки;
- после пойманного флага включается **кулдаун** (`0.5 × минимальный замеренный
  промежуток`): секунды сразу после захвата не опрашиваем - окно так рано не
  откроется.

Итог: ~6 флагов/мин при ~15–22 запросах на флаг и **нуле
rate-limit** - узкий точный залп вместо широкого слепого. Наращивать
конкурентность бессмысленно: флаг на окно один, расписание общее.

## Найденные уязвимости

1. **Нет настоящей аутентификации.** Пароль = `md5(username)`, без секрета - можно
   войти под любым пользователем на любом уровне.
2. **Предсказуемое расписание.** Окна на границах целых секунд - опрос вслепую не
   нужен, достаточно точного тайминга.
3. **Клиентская фильтрация decoy.** Референсный `capture.js` прячет decoy-флаги
   только проверкой `!flag.includes('decoy')` на клиенте.
4. **Флаги глобальные, без привязки к сессии** - нет изоляции пользователей.
